מסמך זה מציג דרכים לשדרג את ה-AI Scout מבוסס נתונים אמיתיים, סטטיסטיקות לפי עמדה, ניתוח וידאו, ורעיונות נוספים.

## 1. הרחבת נתונים אמיתיים (FBref + מקורות)

### 1.1 העשרת FBref — כיסוי רחב יותר

**בעיה:** `enriched_with_fbref: 0` ברוב התוצאות — רוב השחקנים ללא סטטיסטיקות.

**פתרונות:**

1. **אוטומציה של enrich_fbref** — להריץ enrich לאחר כל build דאטאבייס:
   - להוסיף `/build_database` → `enrich_fbref` אוטומטית בסוף
   - או להריץ enrich על ליגות חדשות בזמן בילד

2. **שיפור matching — שם + מועדון + ליגה**  
   FBref משתמש ב־player name + team. ה־matching בין Transfermarkt ל־FBref צריך לעבוד על:
   - שם + מועדון
   - שם + ליגה
   - נורמליזציה של שמות (תווים, דיאקריטיקה)

3. **הוספת ליגות ל־FBREF_LEAGUES**  
   ליגות חסרות כמו:
   - Ligat Haal (ישראל) — כבר יש
   - ליגות נוספות מ־Africa, Asia

4. **שימוש ב־FBref player URL**  
   אם יש קישור ל־FBref פרופיל — לגשת ישירות לסטטיסטיקות במקום לשלוף מטבלאות ליגה.

### 1.2 סטטיסטיקות מתקדמות

| סטטיסטיקה | מקור | שימוש |
|-----------|------|-----|
| **xG (Expected Goals)** | FBref (יש) | איכות סיום, החלטות |
| **xA (Expected Assists)** | FBref | יצירתיות |
| **Progressive passes** | FBref | השפעה התקפית |
| **Pressures** | FBref | אינטנסיביות |
| **Ball recoveries** | FBref | התקפה אחרי איבוד |

**להרחיב:**  
- להוסיף עמודות ל־`enrich_database_with_fbref`  
- להגדיר משקלים לפי עמדה ל־xG, xA

## 2. סטטיסטיקות לפי עמדה — מדויקות יותר

### 2.1 פרופיל עמדה (Position Profile)

```python
# דוגמה: CF (חלוץ מרכזי)
CF_PROFILE = {
    "primary": ["goals_per90", "shots_per90", "goals_per_shot", "xG"],
    "secondary": ["goal_contributions_per90", "fouled_per90", "aerial_won"],
    "penalize": ["tackles_per90"],  # לא רלוונטי לחלוץ
}
```

**לכל עמדה:**  
- להגדיר רשימת סטטיסטיקות ראשיות  
- לחשב ציון מותאם עמדה (Position-Adjusted Score)  
- להשתמש ב־percentiles בתוך אותה עמדה

### 2.2 Percentiles לפי עמדה

במקום להשוות חלוץ לכל השחקנים — להשוות רק ל־CFs:

- חלוץ ב־90th percentile ב־goals/90 — חלוץ קלע
- חלוץ ב־90th percentile ב־tackles — לא רלוונטי

**לממש:**  
- `_compute_position_percentile(player, stat, position)`
- לכל סטטיסטיקה — לחשב percentile רק מול שחקנים באותה עמדה

### 2.3 "Playing Style" מבוסס נתונים

במקום סיווג פשוט — לחשב פרופיל סטטיסטי:

- **Poacher** — goals/90 גבוה, shots_on_target גבוה
- **Target man** — aerial_won, fouled גבוה
- **False 9** — key_passes, progressive_carries גבוה
- **Presser** — pressures, tackles גבוה

**לממש:**  
- `classify_playing_style_from_stats(player)` — מבוסס על סטטיסטיקות בלבד  
- לשמש לסינון ומיון לפי "סגנון"

## 3. ניתוח וידאו קטן

### 3.1 אפשרויות

| אפשרות | עלות | מורכבות | תיאור |
|--------|------|----------|--------|
| **Gemini Vision** | נמוכה (API) | בינונית | העלאת קליפ קצר → Gemini מנתח תנועה, החלטות |
| **Hudl / Wyscout** | גבוהה | גבוהה | API מקצועי — וידאו + אירועים |
| **YouTube highlights** | חינם | קשה | חיפוש היליטים + קישור ל־API |
| **Scoutd.ai** | מנוי | — | פלטפורמה מוכנה |

### 3.2 מימוש מינימלי — Gemini Vision

**רעיון:**  
- המשתמש מעלה קליפ קצר (30–60 שניות)  
- Gemini API מנתח: תנועה, החלטות, טכניקה, חולשות  
- התוצאה מצורפת לדוח הסקאוט

**שימוש:**  
- קישור ל־Gemini Vision API (כבר יש לך skill)  
- Endpoint `/scout_video_analysis` — מקבל קובץ וידאו, מחזיר טקסט ניתוח  

**דוגמה:**  
```
"נותח קליפ של 45 שניות. חלוץ מרכזי — תנועה טובה לחללים, 
סיום חזק ברגל ימין. חולשה: חליפה לאחור איטית. 
מתאים: מועדון שדורש חלוץ קלע עם תנועה."
```

### 3.3 אינטגרציה עם ה־Scout

- להוסיף שדה `video_analysis` ל־`Player`
- לשלב את הניתוח ב־`explanation` / דוח הסופי
- לאפשר העלאת וידאו רק לשחקנים שכבר ברשימה

## 4. שיפורים נוספים

### 4.1 Contract Intelligence

- **חודשים עד חוזה** — כבר חישוב
- **סיכון חופש** — להדגיש שחקנים עם חוזה קצר
- **השוואה לשכר** — אם יש נתוני שכר (מקור חיצוני)

### 4.2 Injury / Availability

- **Transfermarkt** — לפעמים יש injury history
- **מקורות חיצוניים** — API חינמי (כמו injury-api.com) אם קיים

### 4.3 Scouting Report מבוסס נתונים

במקום טקסט גנרי — דוח עם:

- **סטטיסטיקות מרכזיות**  
  - גרפים/percentiles  
  - השוואה לממוצע עמדה  
- **חוזקות**  
  - סטטיסטיקות ב־90th percentile  
- **חולשות**  
  - סטטיסטיקות ב־20th percentile  
- **המלצה**  
  - "מתאים למכבי תל אביב כי: שווי בטווח, גיל 24, G+A/90 מעל 0.5"

### 4.4 A/B Testing

- להשוות תוצאות: "עם FBref" ל־"בלי FBref"
- לבדוק אם משקלים לפי עמדה משפרים את הרלוונטיות
- לאסוף feedback מהמשתמשים (דירוג, "התאים/לא התאים")
