Club Requests, Reimagined

The current requests screen buries critical information inside nested accordions. Here's a proposal to surface data faster, reduce clicks, and let agents scan requests at a glance.

⚠ Pain Points

What's Not Working

Four key issues identified in the current Position → Country → Card accordion design.

🪆

Triple Nesting Fatigue

Position → Country → Card requires 3 expand taps to reach a single request. With 30+ requests across 10 positions, agents spend more time opening/closing than actually reading.

🔍

No Cross-Position Scanning

Can't see all requests from one country at once. If an Israeli club needs a CB and a CM, those requests are separated by sections and there's no way to see them together.

👻

Hidden Match Counts

Matching players count is only visible after expanding a request. Agents can't quickly identify which requests have strong roster matches vs. which need online scouting.

📋

Repetitive Visual Noise

Every request repeats the same chip pattern (Salary, Fee, Age, Foot). Important signals like "EU Only" or special notes get lost in the visual noise of identical chip layouts.

✦ Proposal

Key Design Changes

What changes and why — applicable to both web and Android.

01

Flat Table View (Web) / Flat List (Android)

Replace the triple-nested accordion with a flat, scannable layout. Every request visible as a single row (web) or card (Android). No more expand-to-see-basic-info.

02

Smart Filter Bar

Add a pill-based filter strip at the top: filter by position, country, match status (has matches / no matches), salary tier. Replaces the accordion grouping with user-driven grouping.

03

Match Count Always Visible

Show match count as a colored dot + number directly on every row/card. Green (3+), Orange (1-2), Gray (0). Agents instantly see which requests need attention.

04

Inline Preview of Top Matches

Clicking a row reveals an inline detail panel showing the top 3 matching roster players + top 3 AI scout suggestions side by side — no page navigation needed.

05

Structured Key-Value Grid (Android)

Replace scattered chips with a clean 3-column grid showing Salary, Fee, and Age in labeled boxes. Easier to parse and compare across cards.

06

Board View Alternative (Web)

Offer a Kanban-style board grouped by position as an optional view. Best for position-focused workflows — each column shows all requests for a position with mini-cards.

🖥 Web

Every request as a single scannable row. Click to expand matching players inline.

📋 Summary

All Changes at a Glance

Web — Before → After

  • Nested accordion → flat table with sortable columns
  • No search → full-text search (club, country, contact)
  • No filtering → position + country + match status filter pills
  • Hidden match counts → visible dot + count on every row
  • Separated roster/scout views → inline side-by-side panel on row expand
  • Single view → table + board view toggle
  • No stats overview → summary strip with total, matched, unmatched, countries
  • Scattered chips → structured budget column (salary + fee stacked)

Android — Before → After

  • 3-level accordion → flat card list (one card = one request)
  • No position filter → horizontal scrolling position filter chips
  • Inline text chips → 3-column key-value grid (salary/fee/age)
  • Hidden match counts → stacked avatar preview + count at card bottom
  • No urgency signals → colored position bar on card left edge
  • Notes as truncated chip → dedicated notes block with accent border
  • No quick stats → stats chip strip (open/matched/unmatched/new)
  • "Find online" buried → CTA appears automatically when 0 matches
