📰 News & Rumors – Feature Plan

Men platform · Combined transfer rumours + lower-division football news feed

1 Overview & Goals

A new /news page on the Men web platform that aggregates two data sources into a single, filterable feed:

  • TRANSFER RUMOURS Scraped from Transfermarkt's /geruechte/aktuellegeruechte/statistik — structured player transfer rumour data with clubs, probability %, market value, positions
  • LEAGUE NEWS Scraped from Transfermarkt's per-competition news pages — headlines, excerpts, and metadata for the leagues we actively work with (non-top-5 focus)

Why Transfermarkt for both? It keeps the scraping stack unified (same fetchHtmlWithRetry + cheerio pattern), avoids third-party API keys, and Transfermarkt already has excellent coverage of the exact leagues in LEAGUE_SLUGS.

3 Architecture & File Structure

mgsr-web/src/ ├── app/ │ ├── news/ │ │ └── page.tsx ← News & Rumors page component │ └── api/ │ └── news/ │ ├── rumours/route.ts ← API: scrape TM rumours │ └── league-news/route.ts ← API: scrape TM league news ├── lib/ │ └── transfermarkt.ts ← Add: handleRumours(), handleLeagueNews() ├── lib/ │ └── api.ts ← Add: getRumours(), getLeagueNews() ├── components/ │ └── AppLayout.tsx ← Add nav item: { href: '/news', labelKey: 'nav_news' } └── contexts/ └── LanguageContext.tsx ← Add translation keys (en + he)

API Routes

  • GET /api/news/rumours?page=1 → calls handleRumours(page)
  • GET /api/news/league-news?leagues=ISR1,NL1,TR1 → calls handleLeagueNews(leagues)

Scraping Functions (in transfermarkt.ts)

  • handleRumours(page: number) — Fetch + cheerio parse rumours table, return typed array
  • handleLeagueNews(leagueCodes: string[]) — Fetch news page for each league in parallel (Promise.allSettled), merge + sort by date, return combined array

Caching Strategy

  • Server-side: In-memory cache with 15-minute TTL (same pattern as LEAGUE_AVG_CACHE)
  • Client-side: getScreenCache / setScreenCache for session persistence (same as Releases page)

4 UI Design Decisions

  • Combined feed by default — rumours and news interleaved by date (newest first)
  • 3 filter tabs: "All" | "Rumours" | "News" — toggle between combined and filtered views
  • League filter chips — quick-filter by league (flags + abbreviations)
  • Search bar — filter by player name, club name, or headline text
  • Rumour card design: Player avatar → name/pos → from-club → arrow → to-club → probability badge + market value
  • News card design: Source badge → headline → excerpt → league tag
  • Quick actions on rumour cards: 🔍 View on TM (external link) | ⭐ Bookmark (future: save to Firestore) | 📋 Add to Shortlist
  • Stats bar: Total rumours today / Total news / Leagues covered
  • RTL support — same isRtl pattern, Hebrew translations

5 Implementation Steps (Order)

  1. Step 1: Scraping functions
    Add handleRumours() and handleLeagueNews() to transfermarkt.ts with cheerio parsing + cache
  2. Step 2: API routes
    Create /api/news/rumours/route.ts and /api/news/league-news/route.ts
  3. Step 3: Client API helpers
    Add getRumours() and getLeagueNews() to api.ts
  4. Step 4: Navigation
    Add { href: '/news', labelKey: 'nav_news' } to navItems in AppLayout (men only)
  5. Step 5: Translations
    Add all news_* keys in both en and he blocks
  6. Step 6: Page component
    Build /news/page.tsx with tabs, filters, cards, search, caching
  7. Step 7: Polish
    Loading skeletons, error states, empty states, external link handling
