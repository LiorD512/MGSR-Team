Implementation Notes

Matching Logic

  • Position: Player must have request position in their positions list
  • Age: If request has min/max age, player age must fall within range
  • Salary: Map player market value (e.g. €500K) to salary bands (>5, 6-10, etc.)
  • Transfer fee: Free/Free loan → contract expired; amounts → market value in range

Data Flow

  • ViewModel fetches requests + roster (players)
  • For each request, compute matching players (position + optional age/salary/transferFee)
  • Expose Map<RequestId, List<Player>> or attach to each request in UI state
  • RequestCard receives matchingPlayers and renders expandable section

Recommendation

  • Option A (Expandable) — keeps context on screen, no navigation. Show up to 5 players; if more, add "See all N matches" that opens Option B's bottom sheet.
